# 对标博主策略全景

> 目标: 汇总该小红书博主公开帖、评论区与私信截图中披露的策略信息, 区分其**早期截面多因子阶段**与**后期多策略 rule-based 阶段**。
> 说明: 本文替代旧 `rotation-benchmark.md`。后续不再默认把博主当前体系等同于“单一日频截面排序策略”。

## 一、结论先行

- 博主**并不是一直在做截面多因子排序**。
- 公开信息显示, 她经历了明显的思路切换:
  - 早期/研究期: 日频截面多因子排序, 以 128 日量价窗口为代表
  - 后期/实盘期: 转向 `rule-based` + `trigger-based` 的多策略组合
- 她当前更接近:
  - **12 个子策略 ensemble**
  - 根据不同市场状态切换
  - 日频尾盘底仓 + 1 分钟级 T+0
  - 部分策略混合基本面/另类数据
- 因此, 我们当前的 `Rotation` 不应再被视为“复刻整个目标策略”, 而应视为:
  - 对标其**早期日频选股能力**
  - 或对标其**当前体系中的一个候选子策略**

## 二、信息源整合

### 早期公开信息

- 5 年日频回测基线指标
- 128 日日 K 量价窗口
- 80~500 亿流通市值
- 尾盘交易, 次日卖出, 20 持仓
- 1 分钟高频 T+0 为日内增强

### 后续公开评论 / 私信信息

- 2025 年上半年仍在研究多因子, 实盘也以动量因子为主
- 后来发现“截面多因子排序这一套存在诸多缺陷”, 开始转向 `rule-based`
- 6 月底开始切到新策略, 从 4 个策略逐步扩展到 12 个
- 纯量价策略只有 3 个, 其他策略混合基本面或另类数据
- 另类数据示例包括:
  - 公司 CEO 年龄
  - 年报页数
- 她也明确说过:
  - 不是不做动量
  - 是不做**截面排序**
  - 仍有策略在做动量, 只是 `trigger` 变了

## 三、策略演化时间线

### 阶段 A: 日频截面多因子研究期

这一阶段可视为博主公开表达最完整、也最容易被外界理解成“主策略”的阶段。

| 维度 | 信息 |

### 阶段 B: 认知转折

从评论区与后续交流看, 她对“纯截面多因子排序”的定位发生了变化:

- 多因子这套的主要优点在于:
  - 容量大
  - 抗过拟合
  - 适合团队协作
- 但这些优点对个人做量化未必是核心矛盾
- 她认为:
  - 基于量价的 `rule-based` 在不同市场环境中要稳定盈利很难
  - 因此必须做**多策略组合**来跨牛熊

### 阶段 C: 当前公开可见的实盘体系

| 维度 | 信息 |
|---|---|
| 总体形态 | 多策略组合, 非单模型 |
| 子策略数 | 从 `4` 个扩展到 `12` 个 |
| 切换方式 | 按不同市场状态自动切换 |
| 纯量价策略 | 约 `3` 个 |
| 其他数据 | 混合基本面 / 另类数据 |
| 动量使用方式 | 仍然存在, 但不再采用截面排序表达 |

## 六、她自己对“截面多因子”的最新表述

### 已明确披露的观点

- 截面多因子排序存在诸多缺陷
- 不是不做动量, 而是不再以“截面排序”作为主要表达
- 纯量价 `rule-based` 在不同市场环境中稳定赚钱很难, 容易失效
- 所以需要多策略组合跨越牛熊

### 对散户视角的启发

- 多因子的“容量大、适合团队协作、便于 alpha / beta 分工”这些优点, 更偏机构语境
- 个人做量化时, 可以把 alpha / beta / timing / execution 看成一个整体
- 对个人而言, 比起盲目扩因子, 更重要的是:
  - 是否可兑现
  - 是否能跨市场环境
  - 是否能通过多策略分散单逻辑失效

## 七、对我们项目的重新定义

### 旧定义

- `Rotation` 试图直接复刻“博主整个目标策略”

### 新定义

- `Rotation` 只对标:
  - 博主早期公开的日频截面选股能力
  - 或她当前体系中的一个候选子策略

### 跨设备统一约定

1. `target-strategy-evolution.md` 负责定义**系统级长期目标**。
2. `rotation-next-phase.md` 负责定义 `Rotation` 这条线的**子策略研究目标**。
3. 后续不同设备、不同 agent 若讨论 `Rotation`, 默认应遵循:
   - `Rotation != 博主完整体系`
   - `Rotation = 候选子策略`
   - “对标”默认指向博主早期公开的日频截面基线

### 对我们当前路线的影响

1. `Rotation` 继续做没有问题, 但定位应从“终局复刻”改为“子策略打磨”
2. `core_plus_alpha158(kbar_shape)` 这类增强实验仍然成立, 因为我们本来就在打磨一个强子策略
3. 真正长期要逼近的目标, 已不再是“单一日频排序模型”
4. 长期终局更接近:
   - 多子策略组合
   - 市场状态切换
   - 日频底仓 + 分钟级日内增强
   - 部分混合基本面 / 另类数据

## 八、当前可执行的对标框架

### 短期

- 把 `Rotation` 做成一个真正可兑现的强子策略
- 不再要求它单独承担博主完整体系的全部指标

### 中期

- 扩展到 `3~5` 个低相关子策略
- 引入统一组合层与状态切换框架

### 长期

- 再考虑是否逼近“12 子策略 + 分钟级 T+0 + 混合数据源”的完整体系

## 九、阶段性结论

- 旧 `rotation-benchmark.md` 的最大问题不是“数据错”, 而是**把不同阶段的信息混写成了一个静态目标**
- 现在更合理的 benchmark 结构是:
  - **早期基线**: 截面多因子日频能力
  - **当前体系**: 多策略 `rule-based` + 状态切换 + 部分混合数据
- 从这个角度看, 我们当前方向并没有错, 只是需要把项目目标从“复刻单策略神话”修正为“构建可扩展的多策略体系”
